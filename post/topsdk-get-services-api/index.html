关于商家后台系统对接淘宝API问题汇总

2020-01-15 2 min read # 电商系统开发

最近，经常会遇到很多朋友咨询关于淘宝的订单对接问题，也有一些软件开发的朋友，希望可以在现有系统里集成淘宝API接口，实现淘宝订单处理工作。今天正好有时间，特别写一篇关于这类问题的汇总文章，希望可以帮助到有这方面需求的朋友。

我有淘宝店铺没有天猫店铺，是否可以对接淘宝API？

在最新的商家后台系统对接规则里，已经明确说明：

若您为淘宝集市卖家，抱歉无法为您开启商家接入流程，请您到卖家服务市场挑选您需要的工具/系统。

商家后台系统对接有哪些接入标准？

  1. 必须为公司身份（支付宝账号为公司认证）；
  2. 仅限于具有开发能力的天猫商家；
  3. 商家接入只能供自己店铺所用（自己店铺包括自己的一家或多家店铺）；
  4. 申请商家接入必须入驻聚石塔；
  5. 申请入驻需要在产品说明书中提供自研发系统详细功能模块信息（调用API信息，系统架构，功能截图，自研发软件著作权，自研发源代码）。

调用淘宝API订单，是否必须要入聚石塔？

是的，如果你在塔外调用API接口，获取的订单的地址、收货人等关键信息将会被模糊，导致订单数据下载后，无法正常使用。

一般，入聚石塔后，最好开通订单同步服务，将订单数据、商品数据、退款数据定时推送到同步库，这样，也不会出现调用淘宝API时，出现漏单情况了。

选择什么样的技术方案，可以更快对接企业ERP系统?

企业ERP系统对接淘宝API的主要目标是：

  • 获取淘宝店铺订单
  • 调用淘宝API发货接口，实现自动发货
  • 对接菜鸟电子面单
  • 调用库存同步API，实现线下库存与线上库存同步。
  • 要满足以上目标，完全可以通过在塔内部署订单下载的web服务，完成上述目标。
